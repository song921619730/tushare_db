## 详细修复方案（已通过 Tushare 文档验证）

### 类别 A: symbol_source 错误 — 用 stock_code 调用非股票接口 (7张) — CRITICAL

这些接口当前配置使用 `symbol_source: tushare_stock_basic`（股票代码如 `000001.SZ`），
但 Tushare API 实际需要的是基金代码/指数代码/期货代码。

| # | 接口 | 当前策略 | 当前 symbol_source | 正确 symbol_source | API参数 |
|---|------|----------|--------------------|--------------------|---------|
| A1 | fund_nav | per_symbol_period | tushare_stock_basic | **tushare_fund_basic** | `ts_code`=基金代码(如 165509.SZ), `end_date` |
| A2 | fund_portfolio | per_symbol_period | tushare_stock_basic | **tushare_fund_basic** | `ts_code`=基金代码(如 001753.OF, **必选**) |
| A3 | fund_manager | per_symbol | tushare_stock_basic | **tushare_fund_basic** | `ts_code`=基金代码, `ann_date`, `name` |
| A4 | fund_factor_pro | per_symbol | tushare_stock_basic | **tushare_fund_basic** | `ts_code`=基金代码, `trade_date` |
| A5 | index_daily | per_symbol | tushare_stock_basic | **tushare_index_basic** | `ts_code`=指数代码(如 399300.SZ, **必选**) |
| A6 | fut_weekly_monthly | per_symbol | tushare_stock_basic | **fut_basic** (或直接用pro_bar) | 使用 `pro_bar(asset='FT', freq='W'/'M')` |
| A7 | cb_share | date_loop | — | **cb_basic** | `pro_bar(asset='CB', ts_code='...')` |

**验证来源**:
- fund_nav/doc_id=19: 接受 `ts_code`(基金代码), `end_date`
- fund_portfolio/doc_id=10263: `ts_code`(基金代码, **必选**)
- fund_manager/doc_id=208: `ts_code`(基金代码), `ann_date`, `name`
- index_daily/doc_id=95: `ts_code`(指数代码, **必选**), `trade_date`

**修复步骤** (每个接口):

1. 修改 `config/interfaces/*.yaml` 中的 `symbol_source` 和 `fetch_strategy`
2. 删除旧的无效 sync_state 记录
3. 如果表不存在: bootstrap 建 DDL
4. 重新 plan + backfill

**具体 YAML 修改**:

```yaml
# fund_nav: fund.yaml
fetch_strategy:
  kind: per_symbol          # 改为 per_symbol (每只基金调一次即可)
  symbol_source: tushare_fund_basic  # 改为基金代码

# fund_portfolio: fund.yaml
fetch_strategy:
  kind: per_symbol          # 每只基金返回全部持仓
  symbol_source: tushare_fund_basic  # 改为基金代码

# fund_manager: fund.yaml
fetch_strategy:
  kind: full_once           # 不需要遍历，一次返回所有基金经理
  # 不需要 symbol_source

# fund_factor_pro: fund.yaml
fetch_strategy:
  kind: per_symbol          # 改为 per_symbol
  symbol_source: tushare_fund_basic  # 改为基金代码

# index_daily: index.yaml
fetch_strategy:
  kind: per_symbol          # 改为 per_symbol
  symbol_source: tushare_index_basic  # 改为指数代码

# fut_weekly_monthly: futures.yaml
# 注意: Tushare 没有独立的 fut_weekly/fut_monthly 接口
# 方案A: 使用 pro_bar(asset='FT', freq='W'/'M') + fut_basic 代码
# 方案B: 从 fut_daily 数据 resample 生成周线/月线（推荐）
fetch_strategy:
  kind: full_once
  # 或直接 disabled: true（从 fut_daily resample）

# cb_share: bonds.yaml
fetch_strategy:
  kind: per_symbol
  symbol_source: tushare_cb_basic  # 改为可转债代码
```

### 类别 C: 从未被 plan — 6张 — HIGH

这些接口在 `config/interfaces/` 中缺失 YAML 配置。

| # | 接口 | 表 | Tushare API 参数 |
|---|------|-----|------------------|
| C1 | moneyflow_cnt_ths | tushare_moneyflow_cnt_ths | `ts_code`(N), `trade_date`(N) — 同花顺概念资金流向 |
| C2 | moneyflow_ind_ths | tushare_moneyflow_ind_ths | `trade_date`(N) — 同花顺行业资金流向 (doc_id=343) |
| C3 | moneyflow_ths | tushare_moneyflow_ths | `ts_code`(N), `trade_date`(N) — 同花顺个股资金流向 |
| C4 | stk_week_month_adj | tushare_stk_week_month_adj | 实际不存在; 从 daily 数据 resample |
| C5 | stk_weekly_monthly | tushare_stk_weekly_monthly | **Tushare 中这是两个独立接口: `weekly` + `monthly`** |
| C6 | cn_pmi | — | 手动数据, 暂不需 plan |

**重要发现**: Tushare 的 `weekly` 和 `monthly` 是**两个独立接口**，参数相同 (`ts_code`, `trade_date`, `start_date`, `end_date`)，不存在 `stk_weekly_monthly` 这个接口名。

**moneyflow_ind_ths** (doc_id=343): 输出 `trade_date`, `ts_code`, `industry`, `lead_stock`, `close`, `pct_change`, `net_buy_amount`, `net_sell_amount`, `net_amount`。最低积分: 5000-6000。

**对于 5/min 接口的特殊处理**:

- 建议: 单线程 `special_workers=1` 在非交易时段跑
